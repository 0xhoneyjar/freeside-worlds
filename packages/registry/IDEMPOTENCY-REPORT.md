## What's idempotent (cosmetic-only — terraform plan unaffected)

These differences exist between generated and live `.tf`, but produce **zero terraform-plan changes**:

1. **Banner comments**: generated file declares "Auto-generated by ..."; live files have hand-written historical context (B2 migration notes, mibera-honeyroad rationale). Comments don't drive resource state.
2. **Section dividers**: live files use `# ----- Section -----` style; generator strips these.
3. **Inline rationale comments**: live `world-mibera.tf` has multi-line comments before each block (CPU rationale, health-check rationale, secrets-absent rationale). Generator strips.
4. **Whitespace alignment**: live files align the shared-infra block at 24 chars; generator at 25. `env_vars` block: live uses extra padding for value alignment; generator emits canonical single-space.
5. **Output `description =` field**: generator adds `description = "..."` to outputs that previously had only `value =`. Description is metadata, not resource state.

**These differences are accepted and will converge to canonical form on first migration.**

## What's NOT idempotent (semantic — would change resource state)

### ~~1. score-api~~ — RECLASSIFIED (resolved 2026-04-28 same day)

`score-api` was reclassified per [`freeside-modules-as-installables`](https://github.com/0xHoneyJar/loa-hivemind/blob/main/wiki/concepts/freeside-modules-as-installables.md): it's a service (Hono API impl) and its sealed schemas, not a world. When a future world hits the same friction (e.g., a per-world DB with templated connection string), the v1.1 `templated_value` field on `SecretDeclaration` becomes load-bearing. Filed as follow-up in the path-to-zero-diff section below.

### 2. mibera secret naming convention (MEDIUM IMPACT)

**Live**: `arrakis/${var.environment}/worlds/honeyroad/${each.key}` and resource named `aws_secretsmanager_secret.honeyroad`.

**Generated**: `${local.name_prefix}/worlds/mibera/${each.key}` and resource named `aws_secretsmanager_secret.mibera`.

**Differences**:
- Resource address: `honeyroad` → `mibera` (state migration needed).
- Path segment: `honeyroad` → `mibera` (different secrets path; would create new secrets and orphan old).
- Top-level: `arrakis/${var.environment}/...` → `${local.name_prefix}/...` (these resolve to the same string since `name_prefix = "arrakis-${var.environment}"` minus a slash, but the rendered path differs).

**Resolution path**:
- Option A: `terraform state mv` for resource address + accept new naming. Operator needs to re-load secrets via `scripts/load-honeyroad-secrets.sh` (script reads the live secrets and rewrites under new name).
- Option B: Add `secrets_resource_alias` to manifest (`mibera.yaml` declares `secrets_resource_alias: honeyroad` so generator emits `aws_secretsmanager_secret.honeyroad`).
- Option C: Rename the live resources to match canonical form (one-time migration, then forever-clean).

**Recommended**: Option C. The "honeyroad" naming was a B2-migration artifact (mibera-world repo swapped to mibera-honeyroad mid-cycle). Slug is `mibera`; path should be `worlds/mibera/`. One-time `terraform state mv` + load script update.

### 3. midi secret naming path (LOW IMPACT)

**Live**: `arrakis/${var.environment}/worlds/midi/${each.key}` (already canonical-aligned).

**Generated**: `${local.name_prefix}/worlds/midi/${each.key}`.

**Differences**: Top-level path-segment style only. `arrakis/${var.environment}` resolves to `arrakis-${var.environment}` (close but with `/` vs `-`). Need to confirm `local.name_prefix` definition in loa-freeside.

**Resolution path**: Confirm `name_prefix` value; either align generator output to match `arrakis/${var.environment}` style, or accept the canonical form with one-time secret recreation.

**Recommended**: Inspect `loa-freeside/main.tf` (or wherever `local.name_prefix` is defined) and decide. Likely a minor naming-convention reconciliation.

## Path to zero-diff

To get the existing 4 worlds onto the generator without resource recreation:

1. **schema v1.1 (additive)**: add `templated_value` field on `SecretDeclaration` for future worlds with terraform-computed secret values. Add `secrets_resource_alias` for mibera's `honeyroad` legacy name (or migrate).
2. **Reconcile name_prefix path style**: confirm whether canonical is `arrakis-${env}/worlds/{slug}/{key}` or `arrakis/${env}/worlds/{slug}/{key}`. Update generator to match.
3. **One-time terraform state migration** for mibera (honeyroad → mibera resource rename).
4. **Re-run generator + diff**: cosmetic-only. Adopt as source-of-truth.

Estimated effort: 1 day (was 1-2 with score-api; reclassification removed that work from this scope).
